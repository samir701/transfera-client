    Operations & troubleshooting

    1. Start / stop server

    cd server
./scripts/run.sh          # recommended: frees port 8080 first
# or: ./build/server

# stop: Ctrl+C in terminal (SIGINT → g_keepRunning=false → controller.stop())

# force kill stale processes:
lsof -ti :8080 | xargs kill -9
    server/scripts/run.sh server/src/main_runner.cpp:53-56

    2. Where the server is spawned

    Not spawned as a child by Next.js. You run ./build/server as its own process. The browser only makes HTTP requests to it.

    3. How the server is killed

    MechanismWhat happens
    Ctrl+C (SIGINT)onSignal → loop exits → controller.stop()
    kill <pid> (SIGTERM)Same as SIGINT path if handler runs
    kill -9Immediate termination; no graceful stop; OS closes all FDs
    Destructor~FileController() calls stop()

    4. Multiple servers on port 8080

    Symptom: uploads hang, health times out, UI says API offline.
    Cause: several server processes bound or half-dead on :8080.
    Fix: lsof -ti :8080 | xargs kill -9 then single ./scripts/run.sh.

    7. Health checklist

    1. curl http://127.0.0.1:8080/api/health → {"status":"ok"}
    2. UI shows green “API connected”
    3. Upload returns JSON with numeric port
    4. Download with that port saves file with correct extension
