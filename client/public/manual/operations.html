    Operations & troubleshooting

    1. Local development

    cd server && ./scripts/run.sh
cd client && cp .env.local.example .env.local && npm install && npm run dev
curl http://127.0.0.1:8080/api/health

    2. P2P / quota tests

    cd server
./scripts/test-p2p-local.sh              # maxDownloads=1
MAX_DOWNLOADS=3 ./scripts/test-p2p-local.sh
    Kills stale :8080 listeners before test when KILL_STALE=1

    3. Docker (Render image)

    cd server
./scripts/docker-run.sh
# or: docker build -t transfera-api . && docker run -p 8080:8008 -e PORT=8008 transfera-api

    Logs: TRANSFERA_LOG_HTTP=1, TRANSFERA_LOG_VERBOSE=1 (defaults in Dockerfile).

    4. Multiple servers on :8080

    Symptom: random 500s, “Connection refused”, or second download fails.
    Cause: several server binaries on the same port.
    Fix: lsof -ti :8080 | xargs kill -9 then one ./scripts/run.sh.

    6. Health checklist

    1. curl …/api/health → {"status":"ok"}
    2. UI: green “API connected”
    3. Upload → port + maxDownloads in JSON
    4. Download works; repeat until 404 when quota exhausted
    5. Video / >5 MB rejected in browser before upload
