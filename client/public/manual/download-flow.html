    Download pipeline

    GET /api/download/{invite} — coordinated P2P via socketpair, then HTTP attachment.

    1. Parse invite port

    GET /api/download/(\d+)
parsePortFromPath → 1 .. 65535

    2. Claim invite

    claimInviteForTransfer fails with 404 if:

    • Unknown or already removed invite
    • downloadCount >= maxDownloads
    • Another transfer is in progress on same invite

    3. P2P wire protocol

    Filename: report.pdf\n
<raw file bytes until sender closes connection>

    Implemented in receiveViaLocalP2P: creates socketpair(AF_UNIX), sender thread runs fileSenderHandler, HTTP worker receives on the other FD.

    server/src/services/fileSharer.cpp

    4. After successful transfer

    recordSuccessfulDownload(port, path, remaining);
// downloadCount++
// if count >= maxDownloads → consumeInvite (erase map + delete file)
// else invite stays for more downloads

    Response header: X-Downloads-Remaining: <n>

    5. HTTP response

    Content-Disposition: attachment; filename="..."
X-Filename: ...
X-Downloads-Remaining: 0
Content-Type: application/octet-stream

    CORS exposes these headers to the browser client.

    6. Failure behavior

    Failed P2P (header read error, send failure) releases the claim without incrementing downloadCount — user can retry until quota is exhausted.
