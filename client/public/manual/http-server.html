    3. Start: bind then listen (background thread)

    serverThread_ = std::thread([this] {
    if (!server_.bind_to_port("0.0.0.0", port_)) { /* error */ return; }

    4. Stop: cooperative shutdown

    void FileController::stop() {
    if (!running_.exchange(false)) { join; return; }
    server_.stop();      // unblocks listen loop
    serverThread_.join();
}
    server/src/services/fileController.cpp:150-163

    server_.stop() closes the listening socket so listen_after_bind() returns. join() prevents use-after-free.

    5. httplib thread pool

    Each accepted HTTP connection is handled on the library’s internal ThreadPool (default worker count ≈ hardware concurrency). Your handler code runs on a pool thread, not on serverThread_.

    Implication: handleUpload and handleDownload may run concurrently on different pool threads. Shared state is fileSharer_ map — no mutex today; single-writer per port in practice.
