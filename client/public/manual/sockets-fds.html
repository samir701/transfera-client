    Sockets & file descriptors

    2. P2P server socket creation (step by step)

    const int server_fd = ::socket(AF_INET, SOCK_STREAM, 0);
::setsockopt(server_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

sockaddr_in addr{};
addr.sin_family = AF_INET;
addr.sin_addr.s_addr = htonl(INADDR_ANY);
addr.sin_port = htons(static_cast<uint16_t>(port));

::bind(server_fd, reinterpret_cast<sockaddr*>(&addr), sizeof(addr));
::listen(server_fd, 1);
const int client_fd = ::accept(server_fd, ...);
    server/src/services/fileSharer.cpp:105-151

    Syntax tooltips

    • AF_INET — IPv4
    • SOCK_STREAM — TCP
    • SO_REUSEADDR — reuse port after close
    • htons — port endianness
    • INADDR_ANY — listen on all interfaces

    5. FD leak scenarios

    • Early return without close(sock) — mitigated on error paths in download handler.
    • Multiple stale server processes on :8080 — not FD leak but port conflict; use scripts/run.sh.
    • Detached P2P thread still in accept — holds server_fd until one client connects and transfer completes.
