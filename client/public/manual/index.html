    Transfera — Technical Architecture Manual

    A line-by-line systems guide to the P2P file sharer: Next.js frontend, C++ HTTP API (cpp-httplib), ephemeral TCP file servers, threads, sockets, and file descriptors.

    Audience: Readable by beginners; depth suitable for graduate-level systems review. Hover dotted terms (like this) for inline tooltips.

    What this project does

    Transfera lets a user upload a file through a web UI. The C++ server stores the file, picks a random high port (invite code), and starts a dedicated TCP listener for that file. Another user enters the invite code; the HTTP API connects to the TCP server, pulls the bytes, and returns them as a browser download.

    ┌─────────────────┐ HTTP :8080 ┌──────────────────────┐ │ Next.js client │ ◄──────────────────► │ FileController │ │ (browser) │ /api/upload │ (cpp-httplib) │ │ │ /api/download/N │ │ └────────┬────────┘ └──────────┬───────────┘ │ │ │ invite code N │ spawn detached thread │ ▼ │ ┌──────────────────────┐ └──────── download via HTTP ────►│ FileSharer TCP │ (proxy to 127.0.0.1:N) │ listen :N, send file│ └──────────────────────┘
