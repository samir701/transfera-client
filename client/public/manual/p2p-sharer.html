    P2P FileSharer

    server::service::FileSharer — invite catalog, per-download P2P transfer, download quota.

    1. SharedFile record

    struct SharedFile {
    std::string path;
    std::string downloadName;
    int maxDownloads{1};
    int downloadCount{0};
    bool transferInProgress{false};
};
    server/include/server/service/fileSharer.hpp

    2. offerFile

    int offerFile(filePath, downloadName, maxDownloads = 1);

    Random invite port 49152–65534; stores file metadata. Does not open TCP on that port for HTTP downloads.

    3. receiveViaLocalP2P (HTTP download path)

    1. claimInviteForTransfer — mutex + in-progress flag
    2. socketpair — two connected FDs
    3. Detached sender thread: fileSenderHandler(sendFd, path, name)
    4. Receiver: recvLine (header), recvAllToString (body)
    5. recordSuccessfulDownload — bump count, maybe consumeInvite
    Avoids upload-time TCP races: each download gets a fresh socket pair; invite port is only a lookup key.

    4. consumeInvite

    Removes entry from available_files_, deletes file from disk, logs P2P invite consumed.

    5. startFileServer (legacy / remote)

    Optional TCP listener bound to the invite port for true remote peers. Not used on the normal upload → HTTP download path.
